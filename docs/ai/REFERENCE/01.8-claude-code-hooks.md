## Purpose

Claude Code supports **hooks** — shell commands the harness runs in response
to lifecycle events (session start/end, tool use, etc.). This document
catalogs the hooks configured in `.claude/settings.json` for this repo,
explains the safety contract behind each one, and tells the operator how
to invoke, audit, or disable them.

This is **not** part of the WP/EC governance chain. Hooks are harness
infrastructure: they belong to the Claude Code IDE/CLI, not to the
project's gameplay or governance logic. Their authority comes from
the harness, not from `.claude/CLAUDE.md`.

---

## Active hooks

### `SessionStart` — prunable-branch smoke alarm

**Registered in:** `.claude/settings.json`
**Script:** `scripts/prune-empty-claude-branch.mjs`
**Mode:** `--report` (read-only)

**What it does:** Runs at the start of every Claude Code session. Audits
all local `claude/*` branches against the safety contract below. Counts
how many are *prunable*. If the count is below the threshold (5), prints
nothing — silent pass. If the count meets or exceeds the threshold, prints
a one-line recommendation pointing at the manual cleanup command.

**Why a smoke alarm, not auto-delete:**

- Auto-delete from a `SessionStart` hook would be running inside a fresh
  worktree the user just spawned — deleting from inside that worktree is
  not safely possible (`git worktree remove` cannot remove the worktree
  it's invoked from).
- The destructive action is operator-driven by design. The hook surfaces
  the state; the operator decides when to clean up.
- Conservative threshold (5) means routine accumulation of 1–4 branches
  doesn't generate noise.

## The prune script

### Three invocation modes

```pwsh
# Read-only audit across all claude/* branches in the canonical clone.
# What the SessionStart hook runs.
node scripts/prune-empty-claude-branch.mjs           # default mode = --report
node scripts/prune-empty-claude-branch.mjs --report  # explicit

# Operator-driven bulk cleanup. Never invoked by a hook.
node scripts/prune-empty-claude-branch.mjs --execute

# Per-branch verification gate. Use from inside a worktree, after a PR
# merges, BEFORE running `git branch -D`. PASS / FAIL with exit code.
node scripts/prune-empty-claude-branch.mjs --verify-current
```

### `--verify-current` — per-branch pre-delete gate

Runs against the **current branch** (wherever the script is invoked from
— typically inside a worktree just after its PR merged). Exits 0 on
PASS, 1 on FAIL with operator-actionable diagnosis.

PASS contract (ALL must hold):

1. Current branch name matches `^claude/`
2. Working tree has no uncommitted changes except a modification to
   `.claude/settings.local.json`
3. Branch is in a "merged" state, defined as EITHER:
   a. 0 commits ahead of `origin/main` (true merge / fast-forward), OR
   b. A merged PR exists on origin with this branch as head
      (the squash-merge case; queried via `gh pr list`)

The squash-merge fallback is what distinguishes `--verify-current` from
the bulk-prune modes. Squash-merged branches always show >0 commits
ahead of main (the squash commit has a different SHA than the original
branch tip), so the bulk modes correctly classify them as "blocked"
and leave them for operator review. `--verify-current` resolves them
via the PR-state check.

Required workflow integration: cite this command at Step 7 (draft
close, in `01.0a-wp-drafting-phase.md`) and Step 11 (execution close,
in `01.0b-wp-execution-phase.md`) as the post-merge gate **before**
branch deletion. Failure messages include the specific fix (commit,
stash, merge the PR, install gh CLI, etc.); operator addresses the
failure rather than overriding it.

### Safety contract

A branch is classified as **prunable** if and only if ALL of:

1. Branch name matches `^claude/` (auto-spawned by Claude Code, not an
   operator-authored `ops/`, `infra/`, `spec/`, or `wp-*` branch).
2. Branch is **exactly 0 commits ahead** of `origin/main`.
3. Branch has **no remote tracking ref** on `origin`.
4. If the branch is currently checked out in a worktree: the worktree
   has either no uncommitted changes, or only a modification to
   `.claude/settings.local.json` (the per-worktree Claude harness state).

Branches that fail any condition are classified as **blocked** and never
touched by `--execute`. The blocked set is reported but requires manual
review.

### What this script does NOT handle

- **Merged-via-squash branches** (content on `main` but `commits_ahead > 0`).
  Catching these needs file-content equivalence checks; that's a separate
  audit run periodically by the operator. The big cleanup pass on
  2026-05-11 was an example.
- **Branches with a remote tracking ref.** A pushed branch might be in
  active review. The operator decides PR-or-abandon manually.
- **Worktrees with untracked files or non-harness modifications.** Even
  if the branch is empty, real uncommitted work in the worktree triggers
  the blocked classification.

### Manual override

If the operator wants to force-delete something the safety contract is
protecting:

```pwsh
git worktree remove --force "<worktree-path>"   # if attached
git branch -D <branch-name>
git push origin --delete <branch-name>          # if it has a remote
```

The prune script does not provide a `--force` flag. If you want to delete
something the contract is protecting, you should do it manually so the
operator's intent is recorded in shell history rather than masked behind
a flag.

## Disabling the hook

### Per-machine (no commit, no PR)

Add the SessionStart entry to your private user-level settings with an
empty command, or delete the project-level entry from your local checkout.

### Project-wide (commit + PR)

Edit `.claude/settings.json` and remove the `SessionStart` block. Open a
PR with rationale; this is a governance change.

The script itself is independently invokable from the CLI even with the
hook disabled — the hook is just a convenience trigger.

## Operational notes

- The script anchors all git plumbing to the **canonical clone** (the
  first entry in `git worktree list`). Safe to invoke from any worktree.
- Output is empty when the audit is clean; ASCII-only output otherwise
  (no emoji, no ANSI color) so terminal rendering is predictable across
  PowerShell / bash / VS Code's integrated terminal.
- The script makes no network calls. `ls-remote` queries the local
  refspec cache, not GitHub. Refresh with `git fetch --prune origin`
  beforehand if you want fresh remote state.
- `REPORT_THRESHOLD` is hard-coded at 5. Adjust in the script if your
  tolerance differs. Recording the new value here when changed is
  appropriate but not mandatory — it's a tuning constant, not a contract.

## Authority

This file documents harness wiring. It carries no veto over WP/EC
execution; it does not gate commits, builds, or PRs. The Claude Code
harness alone decides whether a hook fires; the operator alone decides
whether to act on the script's output.

If the hook ever misbehaves (false positive, false negative, or refuses
to run), the operator may delete `.claude/settings.json` without
governance consequences. The catastrophe surface is bounded.
