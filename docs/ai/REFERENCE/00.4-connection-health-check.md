## Code Style Mandate

All output must follow `00.6-code-style.md`. Key rules for this prompt:

- **The script is a linear top-to-bottom sequence** — checks are organized into
  named async functions (`checkTools()`, `checkEnvironmentVariables()`,
  `checkConnections()`, etc.) called in order from `main()`. No class, no runner
  factory, no middleware chain.

- **Each check is its own named async function** — `checkPostgresConnection()`,
  `checkBoardgameioServer()`, `checkCloudflareR2()` — not an array of config
  objects iterated dynamically:
  ```js
  // BAD — dynamic iteration hides what is being checked
  const checks = [
    { name: 'PostgreSQL', fn: checkPostgres },
    { name: 'R2', fn: checkR2 },
  ];
  for (const check of checks) { await runCheck(check); }

  // GOOD — each check called directly, visible in the call stack
  await checkPostgresConnection();
  await checkBoardgameioServer();
  await checkCloudflareR2();
  ```

- **Result objects are built explicitly** — each check returns
  `{ passed: true, message: '...' }` or `{ passed: false, message: '...', remediation: '...' }`
  built inline, not from a `makeResult()` helper.

- **No abbreviations** — `toolName`, `resolvedPath`, `versionString`,
  `remediationMessage`, not `name`, `path`, `ver`, `fix`.

- **Full-sentence output messages** — every printed line tells the developer
  exactly what was checked, what the result was, and what to do if it failed.

- **`// why:` comments** required on: the concurrent connection check pattern,
  the `process.env.APPDATA` rclone path, the secret-masking logic, and the
  boardgame.io CJS entrypoint check.

- **PowerShell script uses `Get-Command`** — not `where.exe`, not `which`.
  Each section is clearly labelled. Output is full sentences.

## Deliverables

### 1. Health check script (`scripts/check-connections.mjs`)

A standalone Node.js ESM script that:

1. Loads `.env` using Node.js v22's built-in `--env-file` flag (invoked via
   `package.json` script — no `dotenv` package import needed in this file)
2. Runs independent checks concurrently; runs env-dependent checks (DB, HTTP)
   after the env var check passes
3. Never throws unhandled exceptions — wraps every check in try/catch
4. Prints a live status line as each check completes (not batched at end)
5. Prints a final summary report
6. Exit code 0 if all checks pass; exit code 1 if any check fails — as a plain
   `if/else` block, not a ternary

Each check is its own named async function with a JSDoc comment.
No check function exceeds 30 lines — break longer checks into named sub-steps.

#### Note on dotenv-cli vs `node --env-file`

Two distinct tools, different use cases:

- **`node --env-file=.env`** — Node v22 built-in flag. Used in `package.json`
  scripts to inject `.env` variables. No package install required.
- **`dotenv-cli`** (`dotenv` binary) — A global CLI tool. Used in scripts where
  `--env-file` is unavailable (e.g., `pnpm exec` chains, PowerShell scripts).
  Install with: `npm install -g dotenv-cli`

Both are checked. The health check itself uses `--env-file`. Other project
scripts (migrate, seed, validate) may use `dotenv-cli` as fallback.

#### Check implementations

**`checkNodeVersion()`**
- Read `process.version`
- Fail if major version < 22
- Remediation: `Install Node.js v22+ from https://nodejs.org`

**`checkPnpmVersion()`**
- Run `pnpm --version` via `child_process.execSync`
- Fail if not found or version < 8
- Remediation: `Run: npm install -g pnpm`

**`checkDotenvCli()`**
- Run `dotenv --version` via `child_process.execSync`
- Fail if not found — dotenv-cli is required for scripts that cannot use `--env-file`
- Fail if version < 16.x
- Verify `.env` is syntactically parseable: run `dotenv -e .env -- node -e ""`
  and check for non-zero exit code (catches BOM encoding, unquoted special chars)
- Remediation: `Run: npm install -g dotenv-cli`

**`checkBoardgameioPackage()`**
- Read `node_modules/boardgame.io/package.json` using `fs.readFileSync`
  (do not shell out — faster and more reliable)
- Fail if file does not exist — `pnpm install` has not been run
- Verify version is `0.50.x` — fail if major.minor is not `0.50`
- Verify server entrypoint exists: `node_modules/boardgame.io/dist/cjs/server.js`
  ```js
  // why: boardgame.io ships its server module in CommonJS format even though
  // this project is ESM-only. The CJS entrypoint is what boardgame.io exposes
  // for server-side use. Its presence confirms the package installed correctly.
  ```
- Remediation: `Run: pnpm install` or `Run: pnpm add boardgame.io@0.50`

**`checkZodPackage()`**
- Read `node_modules/zod/package.json` using `fs.readFileSync`
- Fail if file does not exist
- Remediation: `Run: pnpm add zod`

**`checkDotenvFile()`**
- Verify `.env` exists at the project root
- Verify it is NOT byte-for-byte identical to `.env.example`
- Parse the file and flag values matching placeholder patterns:
  `your-*`, `change-me`, `REPLACE_*`, `<...>`, or empty string
- Fail if `.env` is missing entirely
- Warn (not fail) if placeholder values are found — developer may be in progress

**`checkRequiredEnvironmentVariables()`**
- Check each var in `REQUIRED_VARS` is present in `process.env` and non-empty
- Group output by service — written out as explicit named groups, not iterated
  from a flat array
- Fail if any required variable is missing, naming the specific missing variable

**`checkPostgresConnection()`**
- Connect using `DATABASE_URL` via `pg` pool
- Run `SELECT current_database(), version()`
- Verify database name matches `process.env.EXPECTED_DB_NAME`
- Disconnect cleanly after check
- Fail on connection error — include the pg error code and a full-sentence hint
- Timeout: 5 seconds

**`checkBoardgameioServer()`**
- GET `${GAME_SERVER_URL}/health`
- Expect HTTP 200 and JSON body `{ status: 'ok' }`
- Fail if unreachable, wrong status, or response > 5 seconds
- Remediation: `Is the Render service running? Check https://dashboard.render.com`

**`checkCloudflareR2()`**
- GET `${R2_PUBLIC_URL}/metadata/sets.json`
- Expect HTTP 200 and `content-type: application/json`
- Do not parse body — verify reachability and content type only
- Timeout: 5 seconds

**`checkCloudflarePages()`**
- GET `${CF_PAGES_URL}` (value from env)
- Expect HTTP 200
- Timeout: 5 seconds
- Remediation: `Check Cloudflare Pages dashboard for deployment status`

**`checkGithubReachability()`**
- GET `https://api.github.com/repos/barefootbetters/legendary-arena`
  (unauthenticated — public repo)
- Expect HTTP 200 and `{ name: 'legendary-arena' }`
- Run `git remote get-url origin` via `child_process.execSync`
- Verify local remote URL matches the expected GitHub URL
- Fail if remote is misconfigured or GitHub is unreachable

**`checkRclone()`**
- Verify rclone config exists at:
  `path.join(process.env.APPDATA, 'rclone', 'rclone.conf')`
  ```js
  // why: rclone on Windows stores its config under %APPDATA%\rclone\rclone.conf,
  // not ~/.config/rclone as it does on Linux/macOS. Hardcoding a username path
  // would break on any machine with a different Windows user account name.
  ```
  Warn if config file is missing before attempting CLI commands
- Run `rclone version` via `child_process.execSync` — verify binary on PATH
- Run `rclone lsd r2:` to list the bucket root
- Verify output contains a `metadata` folder and at least one set folder
- Fail if rclone not found, config missing, or cannot list bucket
- Remediation: `Run: rclone config  (see docs\rclone-setup.md)`
