## 4. New Contract / Abstraction — `services/authToken.ts`

The seam future fetcher/auth WPs consume:
- `registerAuthTokenReader(fn)` — called ONCE from `App.vue` after Pinia is created; wires `() => authStore.token`.
- `readAuthToken()` — returns the registered reader's value or `null` (pre-registration / tests). Plain-module, synchronous — preserves the fetchers' synchronous-getter contract.
- `buildLiveRequestOptions(token)` — the **SOLE** producer of live-fetch headers (`{ Accept, Authorization: Bearer <token> }`, no `credentials`).
- `handleMissingAuthToken(cacheRef, warnOnceSet)` — the **SOLE** fail-silent path (one-shot DEV warn keyed in the set; returns the prior cache/sentinel; fires no request).

Design note: on a cache **miss** with a null token the fetcher returns a fresh sentinel **without seeding the cache**, so a later call after the operator signs in retries with the now-present token (no poisoned sentinel). This is stricter than the literal "return cacheRef.value" but matches "same shape as the missing-URL case" and is the correct non-poisoning behavior.

## 5. Execution Reconciliations (No Scope Change)

1. **`buildLiveRequestOptions` grep recalibrated.** The WP/EC §After-Completing expected the count to be `3`. WP §F mandates an **inline** `fetch(url, buildLiveRequestOptions(token))` at every fire path, so the real count is **9** (1 import + N fetch sites per file: 4 analytics + 2 sweep + 3 triage). The expected `3` miscounted (3 files × 1 delegation, ignoring multiple fetcher functions + the import line). The real invariant — delegation present in all three files, `credentials:'include'` **0**, zero inline header literals — holds; WP §F's explicit inline instruction wins over the derived count.
2. **`// why:` comment self-trip (recurrence of `feedback_grep_gate_comment_self_trip`).** The layer-leak grep expects `0` matches for `apps/arena-client`/`apps/server` in `hankoClient.ts`; my required local-copy `// why:` comment named those exact paths to explain the boundary, self-tripping the grep at `2`. Paraphrased to drop the `apps/` path prefix (the comment still conveys "local copy of the arena-client wrapper, not a cross-app import"); the grep now reads a true `0`. This is the same comment-vs-grep class that bit WP-226/233/236/239/240.
3. **`ref(object).value` is a reactive proxy.** A first-draft `authToken.test.ts` asserted `handleMissingAuthToken(...) === sentinel` (the raw input); `ref(obj).value` returns a reactive proxy, not the raw object, so the assertion compares against `cacheRef.value` instead. (The fetcher tests use structural `deepEqual`, which is why they were unaffected.)
